## Mr.Stack — 맥북을 덮어도 일하는 Claude Code

> Claude Code + Telegram = 24시간 상시 대기 AI 개발 파트너

---

### 한줄 소개

Claude Code를 텔레그램으로 확장해서, 선제 알림 + 영구 메모리 + 일일 코칭을 제공하는 macOS 전용 오픈소스 AI 비서입니다.

---

### 만든 배경

Claude Code를 매일 쓰면서 계속 같은 불편함을 느꼈습니다.

터미널을 닫으면 세션이 끝나고, 내일 다시 설명해야 합니다. 자리를 비우면 작동이 멈추고, 돌아와서 맥락을 다시 잡아야 합니다. AI가 나를 얼마나 잘 돕고 있는지, 내가 AI를 얼마나 잘 쓰고 있는지 피드백도 없습니다.

"Claude가 터미널 밖에서도 동작하면서, 내 작업 맥락을 계속 기억하고, 필요할 때 먼저 말을 걸면 어떨까?"

이 질문에서 시작했습니다.

---

### 핵심 기능

**1. 텔레그램 AI 비서**
텔레그램 메시지 하나로 Claude에게 지시합니다. 텍스트, 이미지, 음성, 파일 전부 처리. 맥북을 덮어도 24시간 동작합니다.

**2. Jarvis 모드 (선제 알림)**
5분마다 맥북 상태를 스캔하고, 상황에 맞춰 먼저 텔레그램으로 알림을 보냅니다.
- "3시간째 코딩 중 — 쉬어가세요"
- "배터리 12% — 저장하세요"
- "돌아오셨네요. feature/auth 브랜치 작업 중이었습니다"
- 딥워크 모드에서는 긴급 외 알림 전부 차단

**3. 영구 메모리**
3시간마다 대화를 분석해서 의미있는 정보만 영구 저장합니다. 프로젝트, 사람, 결정, 선호도를 기억하고 다음 세션에 맥락을 이어갑니다. "어제 작업하던 거 이어해줘" → 정확히 이어갑니다.

**4. 일일 코칭 리포트**
매일 밤 텔레그램으로 생산성 리포트가 옵니다. 생산성 점수, 잘한 점, 개선 포인트, 주간 트렌드를 데이터 기반으로 분석합니다. 스크럼 마스터가 매일 밤 나만을 위해 회고해주는 느낌.

**5. 9개 자동화 작업**
아침 브리핑, 저녁 요약, 주간 회고, 캘린더 리마인더, 메모리 동기화, GitHub 알림, Notion 연동 등 9개 cron job이 자동 실행됩니다.

---

### 기술 스택

- Python 3.11+ / claude-code-telegram 기반
- macOS 전용 (osascript, pmset, sysctl 활용)
- 데이터: Markdown + JSONL + SQLite (100% 로컬)
- MCP 연동: Google Calendar, Notion, GitHub, Playwright
- 모델 라우팅: Haiku(단순 조회) / Sonnet(분석) / Opus(심층 회고)

---

### 배운 점

**AI를 "도구"에서 "동료"로 바꾸는 핵심은 기억과 선제성이었습니다.** 기억이 없으면 매번 새로운 사람이고, 선제성이 없으면 결국 수동으로 호출해야 합니다. 이 두 가지를 추가하니 사용 경험이 근본적으로 달라졌습니다.

데이터 레이어는 의도적으로 단순하게 유지했습니다. Markdown과 JSONL이면 충분합니다. 복잡한 DB 없이도 1년치 데이터가 15MB 이내로 관리됩니다.

가장 효과적이었던 기능은 코칭 리포트입니다. "AI를 잘 쓰고 있는가"에 대한 피드백 루프가 만들어지니, 지시 품질이 눈에 띄게 올라갔습니다.
